  • interface IHierarchy ;
    • The Logger hierarchy Interface. We use this to break the interdependency between a couple of modules

  • char[] getName ();
    • Return the name of this Hierarchy

  • char[] getAddress ();
    • Return the address of this Hierarchy. This is typically attached when sending events to remote monitors.

  • void context (Context context);
    • Configure a context

  • Context context ();
    • Return the configured context

  • interface Context ;
    • Context for a hierarchy, used for customizing behaviour of log hierarchies. You can use this to implement dynamic log-levels, based upon filtering or some other mechanism

  • char[] label ();
    • return a label for this context

  • bool isEnabled (ILevel.Level setting, ILevel.Level target);
    • first arg is the setting of the logger itself, and the second arg is what kind of message we're being asked to produce
